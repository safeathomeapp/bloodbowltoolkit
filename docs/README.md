# Docs Guide

This folder intentionally contains more material than should be read at session start.

Use the docs by hierarchy.

## Startup Docs

These are the only canonical startup docs:

1. `docs/SESSION_BRIEF.md`
2. `docs/ARCHITECTURE_CANON.md`

If a future session needs broad project context, start there.

## Operational Doc

- `docs/CODEX_SESSION_OPERATIONS.md`

Use this for server discipline, persistence caveats, and testing/communication rules.

## Reference Docs

These are important, but only task-specific:

- `docs/architecture/`
- `docs/rules_references/`
- `competition_workflow_questions_and_rewrite_guidance.md`
- `blood_bowl_toolkit_codex_master_brief.md`

Important active implementation reference:

- `docs/architecture/2026-05-28_codebase_reuse_map.md`

## Historical Docs

- `docs/session_notes/`
- `docs/roadmap/`

These exist for handoff, audit trail, and historical reasoning.

They are not mandatory startup reading.

## Working Rule

If documentation appears to conflict:

1. `docs/SESSION_BRIEF.md` wins for startup/session context
2. `docs/ARCHITECTURE_CANON.md` wins for product and workflow assumptions
3. task-specific rule/reference docs win only for their exact domain
4. session notes are history, not canon
